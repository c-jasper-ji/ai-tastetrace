# Phase 4: Diversity and Artist Radar

## Objective

Reduce repetitive recommendations and make niche artist discovery a measurable product feature.

## Required work

1. Implement maximal marginal relevance or an equivalent reranker.
2. Add artist and album frequency constraints.
3. Add popularity and familiarity controls.
4. Define an emerging-artist signal with provenance.
5. Build artist-level aggregation and similar-artist exploration.
6. Display relevance, novelty, momentum, and bridge evidence separately.
7. Add catalogue-coverage and intra-list-diversity metrics.

## Verification

- no artist exceeds configured caps;
- album duplication is controlled;
- diversity increases when the control increases;
- relevance loss is measured;
- emerging-first mode changes exposure;
- Artist Radar uses real recommendation evidence.
