# Phase 0: Project Guardrails

## Objective

Prepare a controlled, academically defensible repository before adding features.

## Required work

1. Inspect the repository and current app.
2. Update `BUILD_STATUS.md`.
3. Record implementation and data-source decisions.
4. Confirm the fit index is not described as probability.
5. Define prohibited claims for external review data.
6. Define local ports, commands, and environment variables.
7. Confirm secrets, restricted data, model binaries, and build artefacts are ignored.

## Constraints

- No scraping or bypassing access controls.
- No fabricated ratings or performance metrics.
- No optional feature may displace recommendation quality.
- Preserve credential-free demo mode.

## Verification

- repository map matches README;
- no secrets are present;
- all intended commands are valid;
- Phase 1 is identified as next.
