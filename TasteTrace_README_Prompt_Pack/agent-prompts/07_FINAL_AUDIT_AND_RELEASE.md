# Phase 7: Final Audit and Release

## Objective

Fix all product, technical, data, visual, and release issues before public publication.

## Audit

1. Trace every visible score, tag, label, and recommendation.
2. Remove dead controls, placeholders, unsupported claims, and unused code.
3. Run tests, build, dependency audit, and secret scan.
4. Verify desktop and mobile layouts.
5. Confirm setup instructions from a clean clone.
6. Confirm external data attribution and licence notes.
7. Choose and add an open-source licence.
8. Add contribution and issue templates where useful.
9. Configure GitHub Pages or the selected public host.
10. Run the demo script against the release build.

## Completion

Update `BUILD_STATUS.md` to complete only after every acceptance check passes.
