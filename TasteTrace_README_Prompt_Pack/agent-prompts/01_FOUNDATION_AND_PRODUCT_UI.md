# Phase 1: Foundation and Product UI

## Objective

Create or refine the responsive application shell and complete interaction states.

## Required work

- React and TypeScript app shell;
- navigation for Discover, Artist Radar, Taste Profile, Evidence, and Model Lab;
- loading, empty, error, and partial-data states;
- keyboard focus and accessible labels;
- desktop and mobile layouts;
- coherent visual tokens and icon system.

## UX rules

- First screen is the working discovery tool.
- Avoid a marketing hero.
- Keep dense, professional information hierarchy.
- Use cards only for records, evidence, or tools.
- Do not leave decorative controls.

## Verification

- every navigation item opens a working view;
- no horizontal overflow at 390px;
- no console errors;
- type check, tests, and build pass.
