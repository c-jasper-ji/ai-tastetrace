# TasteTrace Demo Script

Target length: 5-7 minutes.

1. **Problem**
   Explain recommendation repetition, popularity bias, weak transparency, and fragmented review evidence.

2. **Catalogue**
   Show the catalogue statistics: tracks, artists, countries, and niche/emerging share.

3. **Taste anchor**
   Keep the default Radiohead, Massive Attack, and James Blake seeds, then search and add one contrasting track.

4. **Discovery brief**
   Use: `More electronic and atmospheric, with a melancholic edge. Prefer niche and emerging artists.`

5. **Diversity controls**
   Show exploration, result diversity, emerging-first, and seed-artist exclusion.

6. **Recommendation evidence**
   Open a result and explain sonic fit, tag overlap, scene proximity, brief match, evidence strength, long-tail value, and set diversity.

7. **Review trail**
   Open the RYM, AOTY, and Pitchfork links. State that the prototype does not copy live ratings.

8. **Artist Radar**
   Show how track results become artist-level discovery, with fit and momentum separated.

9. **Taste Profile**
   Explain the feature centre and controlled blind spot.

10. **Model Lab**
    Change one weight and increase diversity. Show how the top result responds.

11. **Limitations**
    State that features and evidence are prototype values, the catalogue is finite, and production evaluation requires real behaviour data.
